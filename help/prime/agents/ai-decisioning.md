# AI決策

AI Decisioning是Adobe Journey Optimizer B2B Prime背後的智慧層。 AI決策不會將每個分支預先建置為靜態規則，而是持續評估設定檔的內容，包括歷程會籍、參與歷史記錄、意圖分數和即時行為，以決定該人員的下一個最佳動作或路徑。

這種持續評估將協調流程從靜態規則轉變為結果導向的自動化：您不需要預先定義每個條件，而是描述您想要的結果，系統會決定如何為每個人達成目標。

## 功能 {#capabilities}

AI決策由下列功能組成：

| 功能 | 其決定 |
|---|---|
| [歷程流量控制](../marketing/journey-traffic-control.md) | 個人目前應該處於哪個歷程。 當個人符合多個歷程的資格時，歷程流量控制會在他們的設定檔或行為變更時重新路由他們，而不是將他們留在不再適合的路徑上。 |
| [下一個最佳路徑](../marketing/next-best-path.md) | 個人在歷程中最合適的路徑。 您不必為篩選條件進行硬式編碼，而是會以自然語言描述意圖，而系統會適時將每個人路由至最相符的路徑。 |
| [傳送時間最佳化](../marketing/email-send-time-optimization.md) | 每位收件者的最佳傳送視窗，根據歷史參與度，而非每個人的固定排程。 |
| 內容關聯 | 根據內容摘要自動產生之個人化電子郵件變體，以在歷程中作為單一個人化電子郵件資產使用。 _即將推出。_ |
| [Brand Concierge](https://experienceleague.adobe.com/zh-hant/docs/brand-concierge/content/home){target="_blank"} | 即時對話路由和回應，例如聊天和即時協助。 Brand Concierge需要其他產品權益。 |

其中部分功能可解決相同歷程中的不同問題。 歷程流量控制決定當多個歷程競爭同時吸引同一人時，哪個歷程會獲得優先權。 其他功能則決定當個人已進入歷程後會發生什麼事情。

## 分割路徑與次佳路徑 {#split-paths-next-best-path}

[分割路徑](../marketing/split-merge-paths-nodes.md)可讓您使用明確的篩選條件定義歷程分支：如果分數高於臨界值，請將人員沿著一個路徑傳送，否則沿著另一個路徑傳送。 此規則型邏輯不僅精確且可完全稽核，但每個新條件都需要新分支。

[下一個最佳路徑](../marketing/next-best-path.md)節點將AI決策套用至相同問題。 此模型會一起評估參與、角色、產品興趣和funnel階段，而不是固定的臨界值，預測每個人的最佳結果，並自動選取最佳路徑。

## 資料輸入 {#data-inputs}

AI決策會利用下列資料類別：

| 類別 | 範例 |
|---|---|
| 人口統計和第一統計 | 職稱、產業與公司規模 |
| 心理變數 | 潛在客戶分數、急迫性和優先順序 |
| 參與度 | 分數、層級、趨勢、上次活動和管道 |
| 意圖 | 產品或關鍵字目的，分組為高、中或低值區 |
| 人物誌 | 交易角色、職稱和角色 |

## 評估步調 {#evaluation-cadence}

AI決策使用兩個不同的評估排程。 個人的基礎設定檔會在夜間批次上重新計算。 決定本身會使用最新夜間設定檔所指示，即時套用於每次互動。 因此，AI決策的連續部分描述了決策時刻，而不是設定檔的重新整理率。

## 護欄和規則 {#guardrails-rules}

規則僅涵蓋某人明確寫下的條件。 當現實落在該樹之外時（例如混合角色或沒人預期的訊號組合），規則在有人新增分支之前不會執行任何動作。 AI決策會持續為每個人評分，並改為輸出以信賴度為基礎的最佳動作，因此不會將任何明確程式的組合加以處理，而且會隨著看到更多結果而改善，而規則永遠不會這樣做。

規則易於解釋且立即可供稽核，而AI決策則是以信賴度計分，而非決定性。 因此，在下列情況下，規則仍會是較好的工具：

* 絕對不能越過硬性規範遵循邊界，例如抑制選擇退出的接觸。
* 沒有足夠的容量或歷程記錄可供模型學習。
* 每個決定都必須依需求完整說明。

大部分成熟的設定會同時執行：規則作為護欄，而AI決策則會處理兩者之間的所有內容。

## 優點 {#benefits}

* 減少手動歷程複雜度，需維護的規則分支更少。
* 相關度更高的體驗，無需建立數百個規則。
* 提升資格鑑定效率。
* 更快地為每個設定檔確定下一個最佳參與。

**未來範圍：帳戶與購買群組內容**

目前在Journey Optimizer B2B Prime中無法使用。 AI決策計畫從個別設定檔延伸至：

* 帳戶內容 — 作為決策輸入的公司和帳戶層級訊號。
* 購買群組訊號 — 交易中的角色、決策者或從業人員狀態，以及參與購買決策的所有人彙總的參與度。
* 帳戶層級的歷程協調 — 以帳戶或購買群組為單位進行路由和內容決策，由多個相關人員協調歷程流量控制。
